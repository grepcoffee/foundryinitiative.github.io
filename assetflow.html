Features

Everything your team
needs to manage ministry assets

AssetFlow was built from real operational needs — because "I don't know where that projector is" is a problem every church has faced.

📦
Asset Registry
Catalog every item your organization owns — audio gear, vehicles, tools, cameras, furniture, tech hardware — with photos, serial numbers, purchase dates, and valuations.
📄
Check-Out Management
Issue assets to staff or volunteers with a due date and purpose. Get automatic reminders when items are overdue. Keep a complete log of every transaction — who had what, and when.
📷
QR Code Labels
Generate and print QR labels for any asset in seconds. Anyone with the app can scan a code to instantly view asset details, status, check-out history, and maintenance records.
🔧
Maintenance Scheduling
Set recurring service intervals for any asset — annual inspections, oil changes, lens cleanings, battery replacements. Get alerted when service is coming up before it becomes a problem.
📍
Multi-Location Support
Organize assets by location — main stage, storage room, fellowship hall, campus B. Know exactly where every item should be and flag it when it moves unexpectedly.
⚙️
Condition Tracking
Log damage, repairs, and wear over time. Each asset carries a running condition history — so you know when it's time to repair, replace, or retire an item before it fails in the middle of a service.
📊
Depreciation Reports
Understand the real financial picture of your inventory. Generate depreciation reports for stewardship reviews, insurance documentation, or end-of-year financial summaries.
🔔
Email Notifications
Automated alerts for overdue check-outs, upcoming maintenance, and warranty expirations. Borrowers get confirmation emails on check-out and reminders before items are due back.
🔒
Role-Based Access
Granular access control — admins manage the full registry, staff can check assets in and out, and viewers can look up asset details without making changes. Right access for every role.
How It Works

Simple asset management workflow

AssetFlow is built around a clear three-step process your whole team can adopt without training.

01 — CATALOG
Register Your Assets
Add every asset to the registry with photos, serial numbers, purchase info, and location. Print a QR label and attach it to the item — now anyone on your team can pull up full details instantly with their phone.
02 — TRACK
Issue, Monitor, Return
When someone needs gear, check it out to them with a due date. AssetFlow tracks who has what in real time and automatically sends reminders when items are overdue — so nothing quietly disappears for months.
03 — MAINTAIN
Stay Ahead of Service
Set service intervals for every asset that needs them. AssetFlow alerts your team before maintenance is due — so you're scheduling planned downtime, not scrambling when something breaks the morning of an event.
Under the Hood

Built to run anywhere,
owned by your organization.

AssetFlow is self-hosted — your inventory data stays on your hardware. No SaaS subscriptions, no per-user pricing, no vendor lock-in.

🖥️ Infrastructure

  • Runs on any Node.js server — Mac mini, Raspberry Pi, VPS, or cloud VM
  • SQLite database with WAL mode for reliable concurrent access
  • Cloudflare Tunnel support for secure public HTTPS access
  • LaunchDaemon / systemd configuration for auto-start on macOS or Linux

🔗 Integrations

  • QR code generation for printable asset labels (browser-native, no dependencies)
  • Resend email API for check-out confirmations and overdue alerts
  • CSV export for asset lists, depreciation reports, and maintenance logs
  • Barcode scanner support — USB and Bluetooth HID-compatible devices

🔒 Security

  • bcrypt password hashing with configurable rounds
  • Session-based authentication with account lockout after failed attempts
  • Role-based access control enforced server-side on every request
  • Rate limiting on authentication and asset submission endpoints

⚙️ Built With

  • Node.js + Express — fast, lightweight server with minimal footprint
  • better-sqlite3 — synchronous SQLite with zero native dependencies
  • Vanilla JavaScript frontend — no framework, no build step required
  • CSS custom properties for consistent theming across all views
